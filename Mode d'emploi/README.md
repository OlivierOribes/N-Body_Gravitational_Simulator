# Usage Guide — `Projet_Version_V7_norm.cpp`

English write-up of [`Mode_d_emploi_programme_V7_norm.pdf`](Mode_d_emploi_programme_V7_norm.pdf) (original French PDF, April 2025), updated for the current repository layout. See the [top-level README](../README.md) for the general project overview and the quick `make` workflow.

## 1. Purpose

The program simulates the motion of celestial bodies (planets, satellites, etc.) using a 4th-order Yoshida integrator and compares the simulation against NASA's Horizons system. The Horizons data files are already provided in [`Fichiers de données Horizon System/`](../Fichiers%20de%20donn%C3%A9es%20Horizon%20System) — extracting them from scratch is described in [Annex A](#annex-a-extracting-a-new-body-from-a-horizons-dump) below, since it's slow and rarely needed.

The program writes several output files, notably:

- **`history_norms.txt`** — norm of each body's position at every simulation step. This is a debugging artifact (used to double check that the norm differences were computed correctly) and has no use for an external reader.
- **`norm_diff.txt`** — difference between the simulated trajectories and the NASA data for each planet, one line per planet per time step, in the form:

  ```
  Day  Planet  NumNorm  NASANorm  Angle  CurvDev  Diff.
  ```

  To turn this into plottable curves, run it through [`tools/Extraction_diffnorm.cpp`](../Codes/tools/Extraction_diffnorm.cpp) (see [Annex B](#annex-b-splitting-norm_difftxt-per-planet)), which splits it into one file per planet.

## 2. Prerequisites

| Requirement | Minimum version |
|---|---|
| C++ compiler | GCC 7, Clang 5, or MSVC 2017 |
| Eigen library | 3.3.7 |
| OS | Linux, macOS, or Windows (MinGW or Visual Studio) |
| Input | `horizons_results_<planet>.txt` files (provided) |
| Output | Files generated by the program (see above) |

## 3. Getting the required files

- **NASA Horizons files**: one `horizons_results_<Body>.txt` per planet (Mercury, Venus, Earth, Mars, Jupiter, Saturn, Uranus, Neptune) — already included in [`Fichiers de données Horizon System/`](../Fichiers%20de%20donn%C3%A9es%20Horizon%20System).
- **Source code**: [`Codes/Projet_Version_V7_norm.cpp`](../Codes/Projet_Version_V7_norm.cpp).
- Copy the Horizons files next to the compiled executable before running it.

## 4. Compiling

Using the Makefile (recommended, see the [top-level README](../README.md#building-and-running)):

```bash
cd Codes
make v7
```

Or directly, matching the original instructions:

**Linux / macOS**

```bash
# Make sure Eigen is installed (see README) — either system-wide or as a local Eigen/ folder
g++ -W -Wall -march=native -O2 -Wall Projet_Version_V7_norm.cpp -o simulation
```

**Windows (MinGW)**

```bash
g++ -W -Wall -march=native -O2 -Wall Projet_Version_V7_norm.cpp -o simulation.exe
```

**Visual Studio**: create a C++ project and add the Eigen library to the project's include settings.

## 5. Running

1. Make sure the `horizons_results_<planet>.txt` files sit in the same directory as the compiled executable.
2. Run it:
   - Linux/macOS: `./simulation`
   - Windows: `simulation.exe`
3. The program performs the simulation and writes its output files in that same directory.

## 6. Generated files

**`history_norms.txt`**

```
# Sun
0 1.123456
1 1.223456
...
```

**`norm_diff.txt`**

```
Day  Planet   NumNorm       NASANorm      Angle(°)  CurvDev  Diff(%)
0    Mercury  1.000000e-03  1.000000e-03  0.0000    0.0000   0.0000
1    Venus    1.234567e-03  1.234567e-03  0.0000    0.0001   0.0012
...
```

## 7. How the program works

1. **Initialization** — bodies (planets and moons) are initialized with their current positions and velocities from NASA data.
2. **Simulation** — the trajectories are integrated using the 4th-order Yoshida method.
3. **Force computation** — gravity between all bodies is computed at every step using Newton's law of gravitation (matrix method, see the top-level README).
4. **Comparison with NASA data** — after the simulation, the program compares simulated vs. NASA trajectories per planet using `vector<Vector3d>` of reference and simulated positions.
5. **File generation** — results are saved to the output files described above.

## Annex A: extracting a new body from a Horizons dump

Use [`tools/dataextraction.cpp`](../Codes/tools/dataextraction.cpp) to turn a raw Horizons text dump into a clean `horizons_results_<Body>.txt` file of `X Y Z` columns.

1. **Fetch the data from JPL Horizons**
   - Open <https://ssd.jpl.nasa.gov/horizons/app.html#/>.
   - Under **Target Body**, select the body (e.g. Saturn).
   - Under **Time Span**, use the same start/end date for every body (this project used 2025-03-13 as the start date).
   - Choose the text output with Cartesian coordinates ("X = … Y = … Z = …").
   - Click **Generate Ephemeris**, then **Download**. The file is always named `horizons_results.txt`.

2. **Configure the extractor**
   - Place `horizons_results.txt` next to `tools/dataextraction.cpp`.
   - Open `tools/dataextraction.cpp` and edit the line:
     ```cpp
     const std::string outputFile = "horizons_results_Saturn.txt";
     ```
     replacing `Saturn` with the body you're processing (e.g. `Jupiter`, `Uranus`, …).

3. **Compile and run**
   ```bash
   cd Codes
   make tools          # or: g++ tools/dataextraction.cpp -o tools/dataextraction
   ./tools/dataextraction
   ```
   This reads `horizons_results.txt` and produces `horizons_results_<Body>.txt` with three columns: X, Y, Z.

4. **Repeat for each body** — go back to Horizons, download `horizons_results.txt` for the next body, update `outputFile` in the source, recompile, and rerun.

## Annex B: splitting `norm_diff.txt` per planet

[`tools/Extraction_diffnorm.cpp`](../Codes/tools/Extraction_diffnorm.cpp) turns the combined `norm_diff.txt` into one file per planet, ready for Gnuplot.

1. **Input format** — every non-empty line of `norm_diff.txt` has exactly 7 fields:

   ```
   Day  Planet  NumNorm  NASANorm  Angle  CurvDev  Diff.
   ```

   | Field | Meaning |
   |---|---|
   | Day | time step (days) |
   | Planet | planet name |
   | NumNorm | norm of the relative vector (col. 3) |
   | NASANorm | reference NASA norm (col. 4) |
   | Angle | angle between vectors (col. 5) |
   | CurvDev | curvature deviation (col. 6) |
   | Diff. | final difference (col. 7) |

2. **Compile**
   ```bash
   cd Codes
   make tools          # or: g++ tools/Extraction_diffnorm.cpp -o tools/extraction_diffnorm
   ```

3. **Choose which metric to extract** — by default the program extracts column 7 (`Diff.`). To extract a different metric, open `tools/Extraction_diffnorm.cpp` and find the line:
   ```cpp
   outFiles[col2] << col1 << " " << col7 << "\n";
   ```
   Replace `col7` with `col5` (Angle) or `col6` (CurvDev), and update the output filename accordingly (e.g. `"Angle_" + planet + ".txt"` instead of `"Difference_Yoshida_" + planet + ".txt"`).

4. **Run**
   ```bash
   ./tools/extraction_diffnorm
   ```
   This reads `norm_diff.txt` and creates one `<Metric>_<Planet>.txt` file per planet, with two columns: `Day` and the extracted value.

5. **Plot the results** — import these files into Gnuplot using the ready-made scripts in [`Gnuplot Script/`](../Gnuplot%20Script).
